07 — v3 验收

6 条硬指标。4/6 没达 = v3 没完成

  1. 新产品 onboarding < 1 小时(Founder 填 project.yaml → wizard → built)
  2. 每个 agent 每天 ≥ 20 个 idea(Railway cron 自动跑,不靠手动 POST)
  3. Reviewer 每周 ≥ 30 件 approve,全在 dashboard 内完成,不切窗口
  4. ≥ 50% 已 approve 的 draft 到达 published 状态(P2 必须开闸)
  5. 每周 ≥ 1 个 metric(clicks/upvotes/conversions)回流到 engine/hooks.md
  6. 6 产品各有 MCP server + OpenAPI + capability manifest + ≥ 3 catalog 收录
规则: 6/29 周末检查。达成 ≥ 4 → v3 closed,启动 v4 规划。
达成 < 4 → 不做 v4。架构是错的,不是功能不够。停止加新东西,先把 v3 加固到 ≥ 4。
