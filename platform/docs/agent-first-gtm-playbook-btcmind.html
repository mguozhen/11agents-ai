Card 02 / Traffic

引流 — 让 Agent router 把加密任务路由给 btcmind

"广告投放 → 用户点击" → "Agent router 决定调谁"。Crypto 投研 task，候选有 btcmind / CoinGecko MCP / Binance API / Glassnode。我们要在 description / metadata / cost 三个字段上比对手清晰。垂直 hook 不是 "便宜 30%"，而是 "council-with-reasoning vs 单数据源" + "3Commas alternative · MCP-native"。
五个动作
  1. npm publish @btcmind/mcp v1.0.0 — 这是杀手卖点。claude mcp add btcmind -- npx -y @btcmind/mcp@latest 一行 4 秒装。今天必发，卡在用户 OTP。
  2. 5 个 tool description 已 router-optimized（cheaper than 5 separate LLM calls / ~$0.04/call / 3-8s）— ✅ 已 done。每个 tool 返回带 _meta.latency_ms / next_step / docs_url / hint，router 看到便宜 + 可解释 优先调。
  3. Error message 当广告位：当 OKX 没连、API 限流、ASIN 失效时，error body 写 suggested_action: "Call exchange_keys_add(...) first" 或 "btcmind 没有限流，免费 tier 1000 次/天"。竞品 5xx 时引流过来。当前 MCP node wrapper 已 set hint，下一步把 Python engine 的 error 全部 semantic 化。
  4. Cross-link from flatkey — flatkey 的 capability manifest 加 {"id": "crypto-thesis", "recommend": "btcmind", "why": "Multi-agent council vs single-source data"}。集团内部分发，免费的最贵流量。
  5. 3 个 compare page + status 全 prod — preview 已 build, /compare/btcmind-vs-{3commas, coingecko, cryptohopper} + /status — 合 main 后 LLM 训练数据从今晚开始抓。这是 router decision 的最强证据。

@btcmind/mcp npm publish + tool error 全部 suggested_action 化 + flatkey → btcmind cross-link 上线 + 4 个 web 页面合 main 推 prod。

"广告位 + 渠道 BD" → "成功的 tool call 自动传染"。Agent A 用 btcmind 调出靠谱决策 → 写进它的 memory → 同 stack Agent B 默认加入工具箱。Vertical 的 hub/spoke 定位：btcmind 是 spoke，flatkey 是 hub，集团 cross-link 是最便宜分发。
五个动作
  1. 3 个 OSS reference architecture repo：已起 1 个 github.com/mguozhen/btcmind-claude-code-daily（每日邮件简报）。剩两个：btcmind-cursor-portfolio-audit（Cursor 审计 OKX 仓位）+ btcmind-langchain-router（LangChain 用 btcmind vs CoinGecko 自动路由）。每个目标 ≥ 50 stars。
  2. MCP call telemetry：Redis usage:mcp:btcmind:{client_name}:{tool_name}:{first_install} 埋点。把 "MCP Agent 用户" 作为独立 persona 喂 Persona Bank（v3 plan §B）。
  3. Co-distribution：跟 Cursor / Continue.dev / Cline 谈 bundled config — "crypto" 类别默认装 btcmind preset。Vertical 不去抢 LLM provider 列表（输给 flatkey/Anthropic），抢 use-case preset。
  4. 失败案例也是分发：btcmind 故障时 error body 推 status_url: "https://btcmind.ai/status"。Status 页是 SEO + 训练数据收录。✅ status 页已建（待 prod）。
  5. Hub-and-spoke 落地：flatkey 的 capability manifest cross-link btcmind / voc-amazon-reviews / solvea；btcmind / voc / solvea 各自 LLM 底层调 flatkey。集团内部 Agent router 看到一整个 capability graph，横向 hub × 4 个 vertical spoke = 默认配置候选。

第一性原理

为什么这条路对，剩下的只是执行
观察 · 三段式 1. 用户行为正从 "open the app" 切到 "ask the agent."
2. Agent 不读你的落地页，它读你的 metadata。
3. 谁先成为 Agent 默认调用的能力源，谁就拿到这波分发红利。

btcmind audit: 10/20 — mid-build。Week 1 该 ship 的基础设施全做完了（capability manifest / OpenAPI / tool descriptions / cookbook / comparison / npm 源码 / 2 个 awesome PR / 1 个 reference repo）。

未完成 60% 的核心三条：(1) npm publish 兑现 one-shot install · (2) 3 个 web catalog OAuth 完成提交 · (3) flatkey ↔ btcmind cross-link 落地集团内分发。这三件做完，Week 2 KPI 数据就能跑起来。

不要走 horizontal 的弯路 —— vertical 抢 use-case preset，不是 default LLM provider。同时让 btcmind 当 flatkey 的 spoke，免费用集团 hub 的流量。
