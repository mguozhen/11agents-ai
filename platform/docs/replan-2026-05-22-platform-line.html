§ 05 · TECHNICAL DEBT TO CLEAR

平台化路上躲不开的 4 块技术债

① 文件系统 fall-through 残留

CLAUDE.md 已明令「永远不走文件系统」,但 projects/*/agents/*/content-bank/ 还有内容、agent.yaml 可能还有引用。Gate 1 内 grep 一遍 readFileSync/readdirSync 在 api 路由的使用,全删。

② Cost 归集

Anthropic SDK 现在按 token 收费,没有按 agent/product/week 的 ledger。新增 cost_events 表,每次 SDK 调用记 workspace_id, agent_id, model, input_tokens, output_tokens。Dashboard Foundation tab 加成本视图。

③ Reviewer SLO + 升级

没有审核时延约束 → Reviewer 会成最大瓶颈。子 issue 创建 + 24h → 自动 ping Reviewer,+ 48h → 升 Boyuan/Hunter。用 node-cron 已有的 scheduler 加这个 job。

④ 6 product × 11 agent matrix

不是 6×11=66 都跑,每个 product 选 4–6 个 agent 做主推。在 Foundation Dashboard 出一张「product × agent activation matrix」,圈出 active 组合(目标 ≈ 30 个 instance)。
